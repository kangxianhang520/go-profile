# 角色 3:运行时角色(容器用,已经存在,不用新建)

就是第 2 题创建 ECS 任务定义时 AWS 自动生成的 **ecsTaskExecutionRole**
(负责拉 ECR 镜像、写 CloudWatch 日志)。我们的 Go 程序访问数据库用的是
连接串里的密码,不经过 IAM,所以运行时不需要额外权限。

三个角色各司其职,权限互不重叠(这就是"最小权限"考点):

| 角色 | 谁用 | 能干什么 | 不能干什么 |
|---|---|---|---|
| pr-build-role | CodeBuild 构建容器 | 推镜像、写日志、变身部署角色 | 碰 ECS/ALB/DNS |
| pr-deploy-role | 部署脚本(临时变身) | 建/删 ECS 服务、目标组、分流规则 | 推镜像、改 IAM |
| ecsTaskExecutionRole | 运行中的容器 | 拉镜像、写日志 | 部署任何东西 |

攻击面对比:如果不拆角色、CodeBuild 一个角色全包,那么任何能改
buildspec.yml 的人(比如提 PR 的外部贡献者!)都能在构建脚本里删库跑路。
拆开后,构建环境被攻破也只能推个镜像。
